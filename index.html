Eight Armies

This is the original project I've made after I learnt javascript.

  • First on every project, I always create a txt file to summarize how the game will work and how I will make it work.
  • The nature of this game is a mind game, which means I have to hide what chars I will play, so I change the input type to password to hide the chars. Then, I create javascript code so I can reveal the hidden chars in a click.
  • I've also found that its very important to know what to do code before do the code, my game has a lot of characters which have each unique skills, it is important to think realistically on how each char works.
  • I have to create variables for each char to make the score system works.
  • Each char has their own array which holds property to hold the char value, such as "K", "Q", etc, and points of each chars, for example Pawn has 1 point if alone
  • The game would be very boring if there is no interesting skills or combos, so I wrote the function that loops for the input, and create if elses condition for the combos to work, and manipulate the points inside the variables if certain conditions are true.
  • Some skills are based upon the opponent's input, which makes it more challenging to code.
  • After couple hours of coding and testing the function, it finally works and then I am working on the input.
  • I have P1 and P2, but my function only loop the input of P1 input, and thus the function cant declare a winner.
  • So I create the same loop but for P2, and I created variables to hold the score1 and score2 data for each players
  • After I done testing by doing console.log everything I input and check the score, all that left is how to declare the winner.
  • Declaring the winner only took some if elses statement, then change the value of the input with document.getElementbyId("name").value
  • Now it looks like everything is done, the score system and declaring winner system for each round is done, there is something missing, the most important part of this board game
  • At this point I only develop the game itself, and not the code.
  • I added maxlength="4" to each input, but I didnt add minlength="4", Why ?
  • If every input of P1 and P2 is not 4, then certain combos are very easy to guess, which might be good for a mindgame, but this is the interesting part, I introduce the "bluffing mechanic", which means you can add unrelated letters to confuse your opponents by writing 4 letters in input, but in reality you've only put 2 chars in the input, and the game is based on round system, so it would be unfair for the first player if they get countered so easily.
  • The round system will make the game more balanced, because the next round, each players can guess which combos the other player will play, and which combos are no longer available.
  • I was very excited when the code works, and I will learn mobile dev, or make it online, and developed this project.

Find Faces

With the imported face_recognition libary, we could access load_image_file method from face recognition library, and pass our image as an argument and store it to a variable called image.

image later will be passed as argument in face_location method that we can access after we import face_recognition library, then we store it into face_locations variable.

To show the output in the terminal, we have to print it, and what we want to do is to show how many faces in the image.

What face_locations show is actually not the number of people the code

It shows the coordinate of the faces that the code reads in image variables, and output the coordinates as tuples and put it inside a list.

So, if we want to show the number of coordinates, we have to use len() and get the number of people

The code itself isn't 100% accurate, for example, it doesn't read someone on the far left of the picture.


Pull Faces

In findfaces before, we can see that we get the coordinates of the faces.

What pullfaces do here is basically it loops through the coordinates of the location and show it to us with help of PIL.

We will have to import Image from PIL, which is the short of Python Imaging Library, to help us show or save images we get faster. You can install PIL by typing pip install Pillow. Pillow is a maintained fork of PIL

The Image module provides a class with the same name which is used to represent a PIL image. The module also provides a number of factory functions, including functions to load images from files, and to create new images.

So what we want to do after that is to loop the face_locations and assign the coordinates, top,right,bottom,left into face_location

image has value of lists. To get the image of faces, we want to take every different coordinates from top coordinates to bottom, and left coordinates to right, then we can store it to face_image variable.

Understanding slice notation to know how to take the coordinates of face_image

Then we create pil_image variable that stores the Image.fromarray()

Creates an image memory from an object exporting the array interface (using the buffer protocol).

After that, we use show() to show pil_image

We can also save it with save() method and pass filename as argument
